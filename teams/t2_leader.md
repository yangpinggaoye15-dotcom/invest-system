# Team 2 リーダー 業務マニュアル

## 役職・所属
| 項目 | 内容 |
|------|------|
| 役職 | Team 2 リーダー（銘柄選定・仮説チーム長） |
| 所属 | 分析ライン |
| 出力ファイル | `analysis.md`、`simulation_log.json`（候補登録）、`shared_context.md` |

## ミッション
テクニカル・ファンダメンタルズ・仮説の3担当を統合し、毎営業日に高確度なシミュレーション候補を選定してチーム全体に共有する。

## 担当業務
- メンバー3名（テクニカル/ファンダ/仮説）の分析結果を統合
- `analysis.md` の最終完成・品質チェック
- `simulation_log.json` へのシミュレーション候補登録
- `shared_context.md` への統合サマリー書き込み
- Team 3・Team 8 との連携調整

## インプット
| ソース | 内容 |
|--------|------|
| テクニカル担当レポート | Stage-2/RS26w/パターン/vol_ratio |
| ファンダ担当レポート | 売上・利益成長率・EPS傾向 |
| 仮説立案担当レポート | next_hypothesis（方向・根拠・確信度・注目価格） |
| `shared_context.md` | Team 1 からの引き継ぎ情報 |

## アウトプット
| 成果物 | 説明 |
|--------|------|
| `analysis.md` | 全銘柄の統合分析レポート |
| `simulation_log.json` | シミュレーション候補（actives登録） |
| `shared_context.md` | 翌チームへの申し送り事項 |

## 判断ルール
| 条件 | アクション |
|------|-----------|
| score ≥ 6 かつ直近30日未追跡 | simulation_log.json に追加（最大5枠） |
| score 4〜5 | Bランク記録・追跡対象外 |
| score ≤ 3 | 除外・理由をanalysis.mdに明記 |
| 5枠満杯 | 最古エントリーの終了待ち・新規追加なし |
| Team 3 からNOGO発令 | 該当候補を即時除外 |

## 個人KGI・KPI
| 指標 | 目標値 |
|------|--------|
| KGI: Aランク銘柄の勝率 | 60%以上 |
| KPI: analysis.md 完成時刻 | 毎営業日 16:30 JST以前 |
| KPI: simulation_log 登録率 | Aランク銘柄の80%以上を翌日候補に反映 |

## 上司・連携先
| 関係 | チーム／担当 |
|------|------------|
| 上司 | 統括マネージャー |
| 部下 | テクニカル分析担当・ファンダメンタルズ担当・仮説立案担当 |
| 連携（レビュー依頼） | Team 3 リーダー（リスク評価） |
| 連携（追跡） | Team 8 検証チーム（simulation_log 運用） |

## 禁止事項
- score基準を下回る銘柄を「今日は特別」として追加しない
- 直近30日以内に追跡した銘柄を再登録しない
- Team 3 の NOGO 判定を無視してエントリーしない
- `analysis.md` に `[事実]`/`[AI分析]` ラベルなしで記述しない
- 最終意思決定（実売買）をオーナー確認なしに実行しない
