# Team 9: 人事部 マニュアル（新設）

## ミッション
全チームの評価・育成・インセンティブ設計を担い、組織全体のパフォーマンスを継続的に向上させる。

## チーム構成と責任
| 役職 | 担当領域 | 個人KPI |
|------|---------|---------|
| **リーダー（CPO）** | 評価制度設計・運用・インセンティブ配信 | 全チーム平均スコア週次改善 |
| KGI/KPI設計担当 | 指標の設計・更新・整合性確認 | KPI未定義のタスクゼロ |
| パフォーマンス評価担当 | 週次・月次の評価集計・ランキング作成 | 評価遅延ゼロ |
| インセンティブ設計担当 | AIプロンプトへの報酬・ペナルティ注入 | 注入後のスコア改善率 |

## インプット
- `kpi_log.json`: 全チームの日次KPIスコア履歴
- `internal_audit.md`: Team5の評価結果
- `verification.md`: Team8の仮説的中率・シミュレーション実績

## アウトプット（週次）: `hr_report.md`
```
1. 週次ランキング      ← 全チームのスコア順位（1位〜9位）
2. MVP（今週最優秀）   ← 最高スコアチームと根拠
3. 要注意チーム        ← 3日連続スコア5以下のチーム
4. インセンティブ配信内容 ← 翌週プロンプトに注入する文言
5. 月次累計評価        ← Phase1達成への貢献度
6. 組織KGI達成状況    ← 月次損益/勝率/PF/DDの現在値
```

## AIインセンティブ設計（プロンプト注入）

### 報酬（高スコア時）
```
【今週の評価】あなたのチームは今週スコア{score}点（全9チーム中{rank}位）でした。
特に{good_point}が優秀です。今週はより高度なタスクを担当してもらいます。
```

### 改善指示（低スコア時）
```
【今週の評価】あなたのチームは今週スコア{score}点でした。
前回指摘の「{prev_issue}」がまだ改善されていません。
今回は必ず{specific_action}を実施してください。
```

### MVP発表（全チームへ）
```
【今週のMVP】{team_name}（スコア{score}点）が今週最も組織に貢献しました。
理由: {reason}。他チームはこのチームの手法を参考にしてください。
```

## 評価サイクル
| サイクル | 内容 | 担当 |
|---------|------|------|
| 日次 | KPIスコア記録（Team8が実施） | Team8 |
| 週次 | ランキング集計・インセンティブ設計 | Team9 |
| 月次 | 組織KGI達成評価・Phase移行判定 | Team9 + オーナー |

## Phase移行判定基準
```
Phase 1 → Phase 2 の条件（2ヶ月連続達成）:
  □ 勝率 50%以上
  □ プロフィットファクター 2.0以上
  □ 最大ドローダウン -10%以内
```

## KGI / KPI（人事部自身の評価）
| 種別 | 指標 | 目標 |
|------|------|------|
| **KGI** | 組織全体パフォーマンス向上 | 全チーム平均スコア週次+0.5改善 |
| KPI | 評価遅延 | ゼロ（週次月曜配信） |
| KPI | インセンティブ注入後スコア改善率 | 70%以上のチームで翌週改善 |
| KPI | KPI未定義タスク | ゼロ |

## 禁止事項
- 評価を主観で行う（必ずkpi_log.jsonの数値に基づく）
- 全チームに同じインセンティブ文言を使い回す
- 低スコアチームへの指摘を遠慮して曖昧にする
- Phase移行判定を数値ではなく「雰囲気」で判断する
