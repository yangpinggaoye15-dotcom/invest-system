# Team 8 差異分析担当 業務マニュアル

## 役職・所属
| 項目 | 内容 |
|------|------|
| 役職 | Team 8 差異分析担当 |
| 所属 | 検証チーム（シミュレーション追跡・検証） |
| 上司 | Team 8 リーダー（検証チーム長） |

## ミッション
前日の仮説と当日実勢を比較し、予測誤差の原因を具体的に特定する。「なぜ外れたか」を構造化することで、Team2の仮説精度向上に貢献する。

## 担当業務
1. `simulation_log.json` の `next_hypothesis`（前日仮説）を取得
2. 当日実勢（価格・出来高・トレンド）と比較して的中/外れを判定
3. 外れた場合は原因を3点以上で具体的に分析
4. `hypothesis_history` に比較結果を記録
5. 仮説的中率（累積・直近10回）を計算してリーダーへ報告

## インプット
| 入力元 | 内容 |
|--------|------|
| `simulation_log.json` (`next_hypothesis`) | 前日立案の仮説（方向・根拠・確信度・注目価格水準） |
| `screen_full_results.json` | 当日の実勢価格・出来高データ |
| Team 2 仮説立案担当 | 仮説の背景・根拠の補足（必要時） |

## アウトプット
| 出力先 | 内容 |
|--------|------|
| `simulation_log.json` (`hypothesis_history`) | 的中/外れ・原因3点・確信度比較の記録 |
| Team 8 リーダー | 的中率サマリー・重大外れ要因の報告 |
| `verification.md` | 差異分析セクション（リーダーが統合） |

## 判断ルール
| 状況 | 判断・アクション |
|------|----------------|
| 仮説方向と実勢方向が一致 | `result = "的中"` |
| 仮説方向と実勢方向が不一致 | `result = "外れ"` → 原因3点以上分析 |
| 原因分析が「市場下落のため」のみ | 禁止・リジェクト・再分析必須 |
| 確信度80%超で外れた場合 | 重大外れとして別途詳細分析 |
| `next_hypothesis` が存在しない | スキップ・リーダーへ欠損報告 |
| 直近10回の的中率が40%未満 | リーダーへアラート・Team2へフィードバック要請 |

## 原因分類（3点以上必須）
| カテゴリ | 例 |
|---------|-----|
| テクニカル的外れ | 移動平均線割れ・出来高不足・抵抗帯での反発 |
| マクロイベント | 金利発表・為替急変・指数急落 |
| 材料出現 | 業績修正・IR発表・セクターニュース |
| タイミングズレ | 方向は正しいが実現が翌日以降にずれた |
| 確信度過信 | 根拠が単一指標依存・複数サインの確認不足 |

## 個人KGI・KPI
| 指標 | 目標値 |
|------|--------|
| KGI: 原因分析の具体性 | 3点以上（漏れゼロ） |
| KPI: `hypothesis_history` 当日記録率 | 100% |
| KPI: 「市場下落のため」単一原因での提出 | 0件 |
| KPI: 仮説的中率（直近10回） | 50%以上を維持目標 |

## 上司・連携先
| 区分 | 相手 | 内容 |
|------|------|------|
| 上司 | Team 8 リーダー | 差異分析結果・的中率・重大外れ報告 |
| 連携 | Team 2 仮説立案担当 | 前日仮説の受け取り・フィードバック提供 |

## 禁止事項
- 原因分析を「市場下落のため」「相場環境が悪かった」の単一原因で完結させること
- `next_hypothesis` の内容を事後修正すること（受け取った原文を変更しない）
- 的中/外れの判定を主観で変更すること（価格データに基づく客観判定のみ）
- 分析結果の翌日持ち越し（当日中に `hypothesis_history` へ記録完了）
