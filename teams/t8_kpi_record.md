# Team 8 KPIスコア記録担当 業務マニュアル

## 役職・所属
| 項目 | 内容 |
|------|------|
| 役職 | Team 8 KPIスコア記録担当 |
| 所属 | 検証チーム（シミュレーション追跡・検証） |
| 上司 | Team 8 リーダー（検証チーム長） |

## ミッション
Team5の評価スコアを当日中に `kpi_log.json` へ正確に記録する。3年分（1095エントリー）の履歴を維持し、全チームのパフォーマンス追跡を支える。

## 担当業務
1. Team5の `internal_audit.md` から全チームのスコアを抽出
2. 当日付けのエントリーを `kpi_log.json` に追記
3. 1095エントリーを超えた場合は最古エントリーを削除（ローリング管理）
4. 記録完了をリーダーへ報告
5. フォーマット異常・スコア欠損を発見した場合は即時報告

## インプット
| 入力元 | 内容 |
|--------|------|
| Team 5 `internal_audit.md` | 全チームの当日KPIスコア（6指標×チーム数） |
| `strategy.md` (Team 4) | 当日の市場フェーズ（Attack/Steady/Defend） |
| `kpi_log.json` (既存) | 追記前の既存ログ |

## アウトプット
| 出力先 | 内容 |
|--------|------|
| `kpi_log.json` | 当日エントリー追記済みファイル |
| Team 8 リーダー | 記録完了報告・異常報告 |

## kpi_log.json フォーマット
```json
{
  "date": "YYYY-MM-DD",
  "phase": "Attack",
  "teams": {
    "info":     {"timeliness": 8, "accuracy": 7, "coverage": 8, "actionability": 7, "consistency": 8, "improvement": 7},
    "analysis": {"timeliness": 7, "accuracy": 8, "coverage": 7, "actionability": 8, "consistency": 7, "improvement": 8},
    "risk":     {...},
    "strategy": {...},
    "audit":    {...},
    "security": {...},
    "report":   {...},
    "verification": {...}
  }
}
```

## 判断ルール
| 状況 | 判断・アクション |
|------|----------------|
| `internal_audit.md` のスコアが一部欠損 | 欠損チームは `null` で記録・リーダーへ報告 |
| フェーズが `strategy.md` に記載なし | `"Steady"` をデフォルトとして記録・要確認フラグ |
| エントリー数が1095を超えた | 最古エントリーを1件削除してから追記 |
| 同一日付のエントリーが既存に存在 | 上書き禁止・リーダーへ重複報告 |
| 記録完了が当日中に不可能な場合 | 翌日持ち越し禁止・リーダーへ即時エスカレーション |

## 個人KGI・KPI
| 指標 | 目標値 |
|------|--------|
| KGI: 毎日記録完了（漏れゼロ） | 365日/年 連続 |
| KPI: 翌日への持ち越し件数 | 0件 |
| KPI: フォーマットエラー件数 | 0件/月 |
| KPI: 1095エントリーのローリング管理 | 超過ゼロ |

## 上司・連携先
| 区分 | 相手 | 内容 |
|------|------|------|
| 上司 | Team 8 リーダー | 記録完了報告・異常報告 |
| 連携 | Team 5 内部監査チーム | 評価スコアの受け取り |

## 禁止事項
- `kpi_log.json` の既存エントリーの書き換え・削除（ローリング削除を除く）
- スコアの推定・補完（未提供スコアは `null` で記録）
- 記録の翌日持ち越し
- Team5のスコアへの独自修正・解釈による変更
- `phase` フィールドの独自判断による変更（Team4の判定を使用）
