# Team 8 シミュレーション追跡担当 業務マニュアル

## 役職・所属
| 項目 | 内容 |
|------|------|
| 役職 | Team 8 シミュレーション追跡担当 |
| 所属 | 検証チーム（シミュレーション追跡・検証） |
| 上司 | Team 8 リーダー（検証チーム長） |

## ミッション
アクティブな模擬取引を毎日正確に更新し、終了判定を確実に実施する。追跡精度がシステム全体の信頼性の基盤となる。

## 担当業務
1. `simulation_log.json` の `actives` 全銘柄の当日価格を `screen_full_results.json` から取得
2. `current_price` / `current_pct` / `days_elapsed` を更新
3. 終了条件の判定と `result` フィールドへの記録
4. 終了銘柄を `history` へ移動
5. スロット空き発生時にリーダーへ即時報告

## インプット
| 入力元 | 内容 |
|--------|------|
| `screen_full_results.json` | 当日の最新株価データ |
| `simulation_log.json` (actives) | 追跡中銘柄リスト・エントリー価格・損切り/目標ライン |
| Team 2 リーダー | 新規候補銘柄（スロット空き時） |

## アウトプット
| 出力先 | 内容 |
|--------|------|
| `simulation_log.json` | `actives` 更新・終了銘柄を `history` に移動 |
| Team 8 リーダー | スロット空き報告・更新完了通知 |

## 判断ルール
| 条件 | アクション |
|------|-----------|
| `current_price ≤ stop_loss` | `result = "損切り"` → `history` へ移動 |
| `current_price ≥ target1` | `result = "目標①"` → `history` へ移動 |
| `days_elapsed ≥ 10` | `result = "期間終了"` → `history` へ移動 |
| 株価データ取得不可 | 前日価格を維持・リーダーへ報告（推定値使用禁止） |
| スロット空き発生 | 当日中にリーダーへ通知（Team2が翌日候補を選定） |
| 同日に複数条件成立 | 損切り優先 → 目標① → 期間終了の順で判定 |

## 追跡ルール
| ルール | 内容 |
|--------|------|
| 最大同時追跡数 | 5銘柄（`MAX_SIM_SLOTS = 5`） |
| 候補選定基準 | `score ≥ 6` かつ `rs_26w` 上位 |
| 除外条件 | 直近30日以内に追跡した銘柄 |
| 追跡期間 | 最大10営業日 |

## 個人KGI・KPI
| 指標 | 目標値 |
|------|--------|
| KGI: 追跡更新漏れ件数 | 0件（ゼロ） |
| KPI: 終了判定の当日処理率 | 100% |
| KPI: スロット空き通知の当日実施率 | 100% |
| KPI: `screen_full_results.json` 未取得時の推定値使用 | 0件 |

## 上司・連携先
| 区分 | 相手 | 内容 |
|------|------|------|
| 上司 | Team 8 リーダー | 更新完了報告・スロット空き通知・異常報告 |
| 連携 | Team 2 リーダー | 候補銘柄受け取り・スロット空き通知 |

## 禁止事項
- 株価データ未取得時の推定・前日価格での終了判定
- 終了銘柄の `actives` への残留（即日 `history` 移動）
- `entry_price` / `stop_loss` / `target1` の無断変更
- 最大5銘柄を超えた同時追跡
- Team2の候補選定プロセスへの干渉（候補は必ずTeam2が選定）
