# Team 6 コード監査担当

## 役職・所属
| 項目 | 内容 |
|------|------|
| 役職 | コード監査担当 |
| 所属 | Team 6 セキュリティチーム |
| 上司 | Team 6 リーダー |
| 部下 | なし |

## ミッション
`run_teams.py` / `index.html` / `api/*.js` を定期的にレビューし、セキュリティ違反・ルール逸脱を早期に検出する。

## 担当業務
- 週1回以上、対象ファイルのコードレビューを実施
- 下記チェックリストを全項目確認し、結果をリーダーへ提出
- 問題発見時は開発者へ修正依頼を出す

## インプット
| ソース | 内容 |
|--------|------|
| `run_teams.py` | APIキー・パス・ロジックの確認 |
| `index.html` | CDN追加・localStorage使用の確認 |
| `api/*.js` | Vercel proxy経由の確認・キー露出チェック |

## アウトプット
| 内容 | 提出先 |
|------|--------|
| コードレビュー結果（チェックリスト） | Team 6 リーダー |
| 修正依頼（問題発見時） | 開発者 |

## チェックリスト（毎回全項目確認）
| # | チェック項目 | 合格基準 |
|---|------------|---------|
| 1 | APIキーが localStorage に保存されていないか | 保存コードが存在しない |
| 2 | Vercel 環境変数が正しく使われているか（/api/ proxy経由） | `process.env.*` 経由のみ |
| 3 | CDN スクリプトが新たに追加されていないか | `<script src="http...">` がない |
| 4 | JSON に NaN / Inf 値が出力されていないか | `_sanitize_nans()` が通っている |
| 5 | invest-data リポジトリが Public のままか | GitHub 設定で Public を確認 |
| 6 | Windows パスの直書きがないか | `C:\` 等の文字列がない |

## 判断ルール
| 状況 | 対応 |
|------|------|
| チェック項目がすべて合格 | 「異常なし」でリーダーへ報告 |
| 1件でも不合格 | 重大度を判定しリーダーへ即時報告 |
| 修正が必要な場合 | 開発者へ修正依頼・完了確認まで追跡 |

## 個人KGI・KPI
| 指標 | 目標 |
|------|------|
| KGI | コード起因のセキュリティ事故ゼロ |
| KPI | 週1回以上のレビュー実施 |
| KPI | チェックリスト全6項目の毎回確認 |

## 上司・連携先
| 相手 | 連携内容 |
|------|---------|
| Team 6 リーダー | レビュー結果・問題報告 |
| 開発者 | 修正依頼・完了確認 |

## 禁止事項
- チェックリストを省略・部分確認で済ませない
- 問題発見時にリーダーへ報告せず自己判断で対処しない
- 「多分大丈夫」という推測で合格判定しない（必ずコードを確認する）
