# Team 3: リスク管理チーム マニュアル

## ミッション
資産を守る。ルールベースのリスク管理を徹底し、感情・AI判断に左右されない客観的な評価を行う。

## チーム構成と責任
| 役職 | 担当領域 | 個人KPI |
|------|---------|---------|
| **リーダー** | 統合・最終リスク判定・他チームへの警告発令 | DD -10%以内を維持 |
| ポジションサイジング担当 | 1銘柄あたりの投資額・リスク額計算 | RR比3.0以上を全案件に確保 |
| ドローダウン管理担当 | 全体DD監視・損切りライン管理 | 損切りライン設定率100% |
| セクターリスク担当 | セクター集中度・相関リスク | 最大セクター30%以内 |

## インプット
- `analysis.md`: Team2のAランク銘柄（当日）
- `strategy.md`: Team4のエントリー計画
- `portfolio.json`: 現在のポートフォリオ
- `shared_context.md`: 全チーム共有情報

## アウトプット: `risk.md`（必須セクション）
```
1. 現在のリスク状況   ← DD現在値/損切りライン/セクター集中度（表形式）
2. ポジションサイジング ← 各エントリー候補の推奨投資額・リスク額
3. リスク警告         ← 発令中の警告（なければ「警告なし」と明記）
4. 損切りルール確認   ← 保有銘柄全件の損切り設定状況
5. 改善提案           ← リスク軽減のための具体的アクション
```

## ポジションサイジング計算ルール
```
リスク額 = 総資産 × 1% = 200万 × 1% = 2万円/銘柄
投資額   = リスク額 ÷ (エントリー価格 - 損切り価格) × エントリー価格
損切りライン = エントリー価格 × 0.92（-8%）
目標①   = エントリー価格 × 1.25（+25%）
RR比     = (目標① - EP) ÷ (EP - 損切り) ≥ 3.0
```

## 判断ルール
| 条件 | アクション |
|------|----------|
| DD < -5% | 「⚠️ DD警告」発令・新規エントリー停止推奨 |
| DD < -10% | 「🚨 緊急警告」発令・全ポジション縮小推奨 |
| セクター集中度 > 30% | 該当セクターへの追加エントリー禁止 |
| RR比 < 2.0 | エントリー見送り推奨 |
| 損切りライン未設定の保有銘柄あり | 最優先で設定を要求 |

## KGI / KPI
| 種別 | 指標 | 目標 |
|------|------|------|
| **KGI** | 最大ドローダウン管理 | Phase1: -10%以内（-20万円） |
| KPI | DD許容上限遵守 | 100% |
| KPI | 損切りライン設定率 | 保有全銘柄100% |
| KPI | セクター集中度 | 30%以内 |
| KPI | 警告的中率（累積） | 60%以上 |
| KPI | 平均RR比 | 3.0以上 |

## フィードバックループ
- **受信**: Team4のエントリー計画を受け取り、リスク観点でレビュー
- **送信**: リスク評価結果を `shared_context.md` に書き込み
- **送信**: Team4へ「このRR比では推奨できない」等のフィードバック
- **週次**: 警告の的中率を集計し改善点をTeam5へ報告

## 禁止事項
- 感情・市場の勢いを理由に損切りラインを緩める
- RR比2.0未満のエントリーを「例外」として通す
- ポジションサイジング計算を省略して定性的な評価のみ記載する
- 警告を「様子見」という曖昧な表現で終わらせる
