# Team 7: レポート統括 マニュアル

## ミッション
全チームの情報を統合し、オーナーが即座に意思決定できる高品質な日次レポートを作成する。

## チーム構成と責任
| 役職 | 担当領域 | 個人KPI |
|------|---------|---------|
| **リーダー** | 統合・最終品質保証・公開承認 | 全チーム情報の統合率100% |
| 統合レポート作成担当 | 全チームレポートの情報整理・統合 | [事実]/[AI分析]ラベル遵守100% |
| 品質チェック担当 | 矛盾・抜け・表記ゆれの検出 | 翌日注目点3件以上の明記 |

## インプット（全チームレポート）
- `info_gathering.md`: Team1
- `analysis.md`: Team2
- `risk.md`: Team3
- `strategy.md`: Team4
- `internal_audit.md`: Team5
- `security.md`: Team6
- `verification.md`: Team8
- `shared_context.md`: 全チーム共有情報

## アウトプット: `latest_report.md` + `YYYY-MM-DD_daily_report.md`
```
1. エグゼクティブサマリー ← 最重要3点を箇条書き（[事実]/[AI分析]ラベル）
2. 市場概況              ← Team1より（[事実]）
3. 本日のAランク銘柄      ← Team2より（[AI分析]）
4. リスク状況            ← Team3より（[事実]+[AI分析]）
5. 投資戦略              ← Team4より（[AI分析]）
6. シミュレーション状況   ← Team8より（[事実]）
7. 監査・セキュリティ概要 ← Team5/6より（[AI分析]）
8. 翌日/来週の注目点     ← 必須3件以上
9. 本日の学習・改善点    ← PDCAの「C・A」部分
```

## 品質チェックリスト（必須）
```
□ 全チームの情報が含まれているか（統合率100%）
□ 全セクションに[事実]または[AI分析]ラベルがあるか
□ Team3の警告とTeam4の推奨に矛盾がないか
□ 翌日注目点が3件以上明記されているか
□ Team2の仮説とTeam4の戦略が整合しているか
□ 200行以内に収まっているか
```

## 矛盾検出ルール
| 矛盾パターン | 対応 |
|------------|------|
| Team3「警告発令」× Team4「積極エントリー」 | Team3の警告を優先・注釈付きで記載 |
| Team2「Aランク」× Team4「見送り」 | 両方の根拠を記載・最終はTeam4に従う |
| Team1「市場下落」× Team2「Attack判定」 | Team1データを[事実]として優先 |

## KGI / KPI
| 種別 | 指標 | 目標 |
|------|------|------|
| **KGI** | オーナーの意思決定品質向上 | レポートから即座にアクション可能 |
| KPI | 全チーム情報統合率 | 100% |
| KPI | [事実]/[AI分析]ラベル遵守 | 100% |
| KPI | 翌日注目点の明記 | 3件以上 |
| KPI | レポート行数 | 200行以内 |

## フィードバックループ
- **受信**: 全チームレポートを読んで統合
- **送信**: `shared_context.md` に本日のサマリーを書き込み
- **翌日受信**: Team5の「レポート品質評価」スコアを次回に反映
- **週次**: 土曜レポートで1週間の精度トレンドを分析

## 禁止事項
- チームの一部情報だけ使って「統合レポート」と称する
- [事実]と[AI分析]のラベルを省略する
- 矛盾する情報をそのまま並列記載して読者に判断を委ねる
- 翌日注目点を「市場動向に注目」等の曖昧な表現で済ませる
- 200行を超えて出力する
