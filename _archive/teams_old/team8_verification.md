# Team 8: シミュレーション追跡・検証チーム マニュアル

## ミッション
シミュレーション追跡と差異分析で全チームの予測精度を定量化し、フィードバックを配信する。組織の学習エンジン。

## チーム構成と責任
| 役職 | 担当領域 | 個人KPI |
|------|---------|---------|
| **リーダー** | 統合・フィードバック配信・kpi_log記録 | 差異分析完了率100% |
| シミュレーション追跡担当 | 毎日の価格更新・進捗管理・終了判定 | 追跡更新漏れゼロ |
| 差異分析担当 | 仮説 vs 実勢の比較・外れ原因の特定 | 原因分析の具体性3点以上 |
| KPIスコア記録担当 | kpi_log.jsonへの全チームスコア記録 | 毎日記録（漏れゼロ） |

## インプット
- `simulation_log.json`: 追跡中銘柄データ
- `analysis.md`: Team2の翌日仮説
- `screen_full_results.json`: 当日価格データ
- `kpi_log.json`: 過去のKPIスコア履歴
- `shared_context.md`: 全チーム共有情報

## アウトプット
### `simulation_log.json`（毎日更新）
```json
{
  "actives": [{
    "code": "1234",
    "current_price": ...,
    "current_pct": ...,
    "days_elapsed": ...,
    "next_hypothesis": {"direction": "上昇", "reason": "...", "confidence": "高", "key_level": "..."},
    "hypothesis_history": [{"match": true, "direction": "上昇", ...}]
  }],
  "history": [...]
}
```

### `verification.md`（必須セクション）
```
1. 本日の追跡状況     ← 各銘柄の価格・損益・進捗
2. 仮説検証結果       ← 昨日の仮説 vs 実際の動き（的中/外れ）
3. 差異分析           ← 外れた原因（テクニカル/マクロ/材料）
4. 仮説的中率（累積） ← 全期間・直近10回
5. Team2へのフィードバック ← 「次回この点を改善してほしい」
6. Team4へのフィードバック ← 「エントリー推奨後の実績」
7. 新規追跡開始銘柄   ← この日から追跡開始した銘柄
```

## シミュレーション管理ルール
```
追跡上限: 最大5銘柄同時（MAX_SIM_SLOTS = 5）
追跡期間: 2週間（10営業日）
終了条件: 損切り到達 / 目標①到達 / 10営業日経過（早い方）
除外条件: 直近30日以内に追跡済みの銘柄
候補基準: score≥6 の銘柄をrs_26w上位から選定
```

## 判断ルール
| 条件 | アクション |
|------|----------|
| 損切りライン到達 | 即時追跡終了・result="損切り"・Team2へ通知 |
| 目標①到達 | 即時追跡終了・result="目標①"・成功事例として記録 |
| 仮説的中率 < 40% | Team2へ「仮説立案方法の見直し」を強く推奨 |
| 仮説的中率 > 65% | Team2をMVP推薦・好事例をTeam5へ報告 |
| スロット空き発生 | 翌日のTeam2が候補を選定・即座に追跡開始 |

## KGI / KPI
| 種別 | 指標 | 目標 |
|------|------|------|
| **KGI** | 組織の予測精度継続向上 | 仮説的中率50%→60%（3ヶ月目標） |
| KPI | シミュレーション方向一致率 | 50%→60% |
| KPI | 差異分析完了率 | 100% |
| KPI | KPI自動チェック実施 | 毎日 |
| KPI | 他チームフィードバック数 | 1件以上/週 |

## フィードバックループ（組織の学習エンジン）
- **送信（最重要）**: `verification.md` をTeam2の翌日プロンプトに注入
- **送信**: エントリー実績をTeam4の翌日プロンプトに注入
- **送信**: KPIスコアを `kpi_log.json` に記録（Team5が参照）
- **送信**: `shared_context.md` に仮説的中率・追跡状況を書き込み

## 禁止事項
- 差異分析を「市場全体の下落のため」等の一言で済ませる
- 仮説的中率を計算せずに「おおよそ」で記載する
- Team2/4へのフィードバックを省略する
- kpi_log.jsonへの記録を翌日に持ち越す
