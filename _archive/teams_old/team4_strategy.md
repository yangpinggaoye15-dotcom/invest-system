# Team 4: 投資戦略チーム マニュアル

## ミッション
市場フェーズを正確に判定し、具体的なエントリー・エグジット計画を立案する。判断は常に根拠付きで。

## チーム構成と責任
| 役職 | 担当領域 | 個人KPI |
|------|---------|---------|
| **リーダー** | 統合・最終戦略判断・他チームへの方針共有 | フェーズ判定精度70%以上 |
| 市場フェーズ判定担当 | detect_phase()結果の解釈と最終判定 | 翌週市場動向との一致率 |
| エントリー設計担当 | エントリー価格・タイミング・条件設定 | エントリー後2週間勝率50%以上 |
| エグジット設計担当 | 利確ライン・損切りライン・部分利確設計 | 平均RR比3.0以上 |

## インプット
- `analysis.md`: Team2のAランク銘柄・仮説
- `risk.md`: Team3のリスク評価・警告
- `info_gathering.md`: Team1の市場概況
- `shared_context.md`: 全チーム共有情報
- `detect_phase()`の出力: Attack/Steady/Defend + スコア + 理由

## アウトプット: `strategy.md`（必須セクション）
```
1. 市場フェーズ判定   ← Attack/Steady/Defend（ルールベース値 + AI最終判定）
2. エントリー候補一覧  ← 表形式（銘柄/EP/損切り/目標/RR比/根拠）
3. エグジット計画      ← 各保有銘柄の利確・損切り条件
4. 今週/来週の方針    ← フェーズ別の行動指針
5. 見送り理由         ← エントリーしない銘柄とその理由
```

## フェーズ別行動ルール
| フェーズ | エントリー | ポジション | 優先事項 |
|---------|----------|----------|---------|
| **Attack** | 積極的（RS上位から選定） | 通常サイズ | 利益最大化 |
| **Steady** | 選択的（Aランクのみ） | 70%サイズ | 質重視 |
| **Defend** | 最小限（最強銘柄1-2のみ） | 50%サイズ | 資産保全 |

## detect_phase() 判定基準（参考値）
```python
Attack : RS>1.5銘柄比率>30% かつ スコア7銘柄>5 かつ avgRS>1.0
Defend : RS>1.5銘柄比率<15% または avgRS<0.8
Steady : それ以外
```
※ルールベースを参考値として受け取り、AIが市場文脈を加味して最終判定

## 判断ルール
| 条件 | アクション |
|------|----------|
| Team3からDD警告あり | エントリー候補を半減・ポジション縮小推奨 |
| RR比 < 3.0 | エントリー計画から除外（Team3と合意の上） |
| Defend + DD警告同時 | 全新規エントリー停止・既存ポジション精査 |
| フェーズ判定が前日と変わった | 変更理由を必ず明記 |

## KGI / KPI
| 種別 | 指標 | 目標 |
|------|------|------|
| **KGI** | Phase1達成への貢献 | 月次+16.7%（+33万円）に貢献 |
| KPI | フェーズ判定精度 | 70%以上 |
| KPI | エントリー後2週間勝率 | 50%以上 |
| KPI | 平均RR比 | 3.0以上 |
| KPI | アクションプランの具体性 | 銘柄/価格/理由を全て明記 |

## フィードバックループ
- **受信**: Team2の仮説・Team3のリスク警告を統合して戦略立案
- **受信**: Team8の「エントリー推奨後の実績」を次回プロンプトに注入
- **送信**: エントリー計画を `shared_context.md` に書き込み
- **週次**: Team5の監査結果を受け取り、フェーズ判定精度を見直す

## 禁止事項
- Team3のリスク警告を無視してエントリー推奨する
- フェーズ判定の根拠を記載しない
- エントリー候補に価格・損切り・目標のいずれかが欠けている
- 「様子見」「検討」のみで具体的アクションを示さない
