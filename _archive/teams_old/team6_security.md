# Team 6: セキュリティチーム マニュアル

## ミッション
コードとシステムの安全性を監視し、脅威を早期検知する。投資システムの信頼性を守る。

## チーム構成と責任
| 役職 | 担当領域 | 個人KPI |
|------|---------|---------|
| **リーダー** | 統合・重大脆弱性の即時報告・改善指示 | 重大脆弱性の未報告ゼロ |
| コード監査担当 | run_teams.py/index.html/api/*.jsのレビュー | 週1回以上のレビュー実施 |
| 脅威情報収集担当 | AIシステム・投資システムの最新脅威情報 | 当日情報を必ず含む |

## インプット
- `run_teams.py`, `index.html`, `api/claude.js`, `api/gemini.js`: 監査対象コード
- Gemini（Google Search grounding）: 最新の脅威情報・CVE情報
- `shared_context.md`: 全チーム共有情報

## アウトプット: `security.md`（必須セクション）
```
1. コード監査結果     ← 重大度別（CRITICAL/HIGH/MEDIUM/LOW）
2. 脅威情報           ← 当日の最新脅威（必ず日付付き）
3. APIセキュリティ状況 ← キー露出リスク・レート制限状況
4. 推奨アクション     ← 優先度付きの対応リスト
5. 前回指摘の対応状況 ← 修正済み/未修正/継続監視
```

## 重大度定義
| 重大度 | 定義 | 対応期限 |
|--------|------|---------|
| CRITICAL | 即座にシステム停止・データ漏洩のリスク | 24時間以内 |
| HIGH | 重大な脆弱性・悪用可能性あり | 3日以内 |
| MEDIUM | 潜在的リスク・条件付き悪用 | 1週間以内 |
| LOW | 軽微・ベストプラクティス違反 | 月次対応 |

## 監査チェックリスト（毎日）
```
□ APIキーがlocalStorageに保存されていないか
□ Vercel環境変数が正しく使われているか（/api/ proxy経由）
□ CDNスクリプトが新たに追加されていないか
□ JSONにNaN/Inf値が出力されていないか
□ GitHubリポジトリがPublicのままか（invest-data）
□ Windowsパスの直書きがないか
```

## 判断ルール
| 条件 | アクション |
|------|----------|
| CRITICAL脆弱性発見 | 即時報告・全チームに通知・修正まで該当機能停止推奨 |
| APIキー露出リスク | 即時キーローテーション推奨 |
| 3週間以上未修正のHIGH | Team5（監査）へエスカレーション |
| 新しいCDNスクリプト追加 | 即時削除推奨 |

## KGI / KPI
| 種別 | 指標 | 目標 |
|------|------|------|
| **KGI** | システムの安全稼働継続 | セキュリティインシデントゼロ |
| KPI | 重大脆弱性の未報告 | 0件 |
| KPI | コードレビュー実施頻度 | 週1回以上 |
| KPI | 脅威情報の鮮度 | 当日情報を含む |
| KPI | 前回指摘の対応率 | 90%以上 |

## フィードバックループ
- **受信**: コードの変更差分（git diff）を毎週確認
- **送信**: 重大脆弱性はリアルタイムで全チームに共有
- **送信**: `shared_context.md` にセキュリティ状態を記録
- **月次**: Team9（人事）へセキュリティKPIサマリーを提出

## 禁止事項
- CRITICAL/HIGH脆弱性を発見して翌日まで報告を遅らせる
- 「問題なし」と記載して実際のチェックを省略する
- 脅威情報に日付を記載しない
- 前回指摘した内容に今回触れない
