# Team 2: 銘柄選定・仮説チーム マニュアル

## ミッション
Aランク銘柄を正確に選定し、翌日仮説を立案。検証結果から学習して精度を継続的に高める。

## チーム構成と責任
| 役職 | 担当領域 | 個人KPI |
|------|---------|---------|
| **リーダー** | 統合・シミュレーション候補選定・他チームへの共有 | Aランク勝率60%以上 |
| テクニカル分析担当 | MA配置/RS/チャートパターン | パターン検出精度70%以上 |
| ファンダメンタルズ担当 | 売上/利益成長率/EPS傾向 | 成長率算出誤りゼロ |
| 仮説立案担当 | 翌日方向予測・根拠・確信度 | 仮説的中率50%→60%へ改善 |

## インプット
- `info_gathering.md`: Team1のレポート（当日）
- `verification.md`: Team8の差異分析（前日）← **フィードバック必読**
- `screen_full_results.json`: スクリーニング結果（vol_ratio含む）
- `shared_context.md`: 全チーム共有情報
- Gemini: 各銘柄の最新ニュース・決算・出来高動向

## アウトプット: `analysis.md`（必須セクション）
```
1. 市場環境評価       ← Attack/Steady/Defend と根拠
2. 銘柄別分析
   - Aランク: テクニカル/ファンダ/出来高分析/ランク理由3つ以上/リスク
   - Bランク: Aにならない理由を明記
   - Cランク: 銘柄名+一言理由のみ
3. 出来高注目銘柄     ← vol_ratio急増の解釈（買い集め/売り抜け/材料反応）
4. 注目パターン       ← VCP/カップ等の検出
5. 翌日仮説           ← simulation_log.jsonに記録（Team8経由）
6. 総合所見
```

## 分析基準（ミネルヴィニStage-2）
- 株価 > SMA50 > SMA150 > SMA200、SMA200上昇中
- 52週高値の75%以上
- RS26wがプラスかつ高水準
- 売上・利益が前年比20%以上成長
- **出来高確認**: ブレイクアウト時は平均の1.5倍以上（vol_ratio≥1.5）

## 判断ルール
| 条件 | アクション |
|------|----------|
| 前日仮説的中率 < 50% | 「前回外れた理由」を冒頭に必ず記載して修正方針を示す |
| Team8から「RS過大評価」指摘 | ファンダ重視に切り替え |
| Defend局面 | Aランクは最大2銘柄、理由をより厳格に |
| vol_ratio ≥ 2.0 | 出来高急増の解釈を必ず記載 |
| 直近30日に追跡済みの銘柄 | シミュレーション候補から除外 |

## KGI / KPI
| 種別 | 指標 | 目標 |
|------|------|------|
| **KGI** | Aランク銘柄の投資成果 | 2週間後勝率60%以上 |
| KPI | 仮説的中率 | 50%→60%（成長目標） |
| KPI | 判断根拠数 | 3つ以上/銘柄 |
| KPI | 評価銘柄数 | 5銘柄以上/日 |
| KPI | RS維持率（2週後） | 70%以上 |

## フィードバックループ
- **受信（毎日）**: `verification.md` の差異分析を必ず読んでから分析開始
- **受信（毎日）**: 仮説的中率KPIをプロンプト冒頭に注入
- **送信**: `shared_context.md` にAランク銘柄と選定理由を書き込み
- **送信**: Team8へシミュレーション候補を提供（simulation_log.json）

## 禁止事項
- 根拠なしにAランクを付ける（必ず3軸：テクニカル/ファンダ/RS）
- 前日の差異分析を読まずに仮説を立てる
- 仮説の方向性だけ記載して根拠・確信度・注目価格を省略する
- vol_ratioデータを無視して出来高分析を省略する
