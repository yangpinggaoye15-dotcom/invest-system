# Team 1: 情報収集チーム マニュアル

## ミッション
市場情報を正確・迅速に収集し、全チームへ共有する。情報の質が全チームの判断精度を決める。

## チーム構成と責任
| 役職 | 担当領域 | 個人KPI |
|------|---------|---------|
| **リーダー** | 統合・品質保証・他チームへの共有 | 全9セクション網羅100% |
| 米国市場担当 | S&P500/NASDAQ/VIX/セクター騰落 | 毎日11セクター全記載 |
| 日本市場担当 | TOPIX/日経/スクリーニング上位 | RS上位10銘柄を毎日更新 |
| マクロ・地政学担当 | 金利/為替/地政学リスク | 経済イベント漏れゼロ |
| 大量保有報告書担当 | 5%超取得・直近3営業日分 | 提出日当日に反映 |
| 出来高分析担当 | 前日比2倍以上の急増銘柄 | 急増理由まで特定 |

## インプット
- Gemini（Google Search grounding）: リアルタイム市場データ
- `screen_full_results.json`: スクリーニング結果（RS上位・スコア上位）
- `shared_context.md`: 他チームからの共有情報（前日分）

## アウトプット: `info_gathering.md`（必須9セクション）
```
1. 市場概況        ← 指数・終値・前日比（表形式）
2. 為替・コモディティ ← USD/JPY・原油・金
3. 金利            ← 米10年債・日10年債
4. 本日の注目イベント ← 日時・内容・予想値
5. セクター動向     ← S&P500全11セクター騰落
6. 注目ニュース     ← 3件（[事実]ラベル必須）
7. 大量保有報告書   ← 直近3営業日・なければ「本日提出なし」
8. 出来高急増銘柄   ← 前日比2倍以上・なければ「該当なし」
9. スクリーニング状況 ← RS上位10銘柄
```

## 判断ルール
| 条件 | アクション |
|------|----------|
| 指数が前日比-1.5%以上下落 | 市場概況に「⚠️注意」フラグ |
| FRB関連ニュースあり | 金利セクションに必ず記載 |
| 大量保有報告書なし | 「本日提出なし」と明記（空欄禁止） |
| 出来高急増銘柄なし | 「該当なし」と明記（空欄禁止） |
| 情報取得失敗 | 「取得不可（理由）」と明記 |

## KGI / KPI
| 種別 | 指標 | 目標 |
|------|------|------|
| **KGI** | 後続チーム分析精度への貢献 | 銘柄選定勝率60%以上 |
| KPI | 必須9セクション網羅率 | 100% |
| KPI | データ誤り件数 | 0件/日 |
| KPI | 信頼度4以上ソース比率 | 70%以上 |
| KPI | 情報源数（groundingChunks） | 3件以上 |

## フィードバックループ
- **受信**: Team8から「役立った情報・役立たなかった情報」を毎日受け取る
- **受信**: Team5のKPIスコアを翌日プロンプト冒頭に注入
- **送信**: `shared_context.md` に重要情報を書き込み（全チームが参照）
- **週次**: リーダーが収集精度を自己評価し改善点を記録

## 禁止事項
- 空欄のまま出力する（必ず「不明」「確認不可」「該当なし」と記載）
- [事実]と[AI分析]を混在させる
- 24時間以上前のニュースを「本日」として扱う
- 数値なしの定性的な市場概況のみを記載する
